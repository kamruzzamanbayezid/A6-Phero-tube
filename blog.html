Q1: Discuss the scope of var, let, and const:

Var:

Variable declare with var is both function scope and global scope. Var is a global variable. We can access it from anywhere. If we need to re-declare and re-assign a variable then we will use var variable. Using var is not a good practise.

Let:

Variable declare with let is block scope variable. They are accessible only within the block (enclosed by curly braces). Although we can re-assign it but we can not re-declare it. Use let instead if var

Const:

Variable declare with const is also block scope variable. The Value of const variable is fixed. We can not re-declare and re-assign it. The value of const variable array and object's property or element can be modified. Use const instead of var and let when your values are fixed.

Q2: The use cases of null and undefined:

Null:

null as an empty container or a placeholder for nothing. In javaScript null is an intentional way of saying 'there is nothing here' or 'this doesn't have a value'.It can be use to indicate something should be empty or doesn't exist. There can use a null value to saying explicitly there's no value here. null can be use to reset a variable or object property.

Undifined:

    When a variable is declared but is not assigned a value than it automatically set the value to undefined. It also given undefined when the assign value is not meaningful. If a function is called with some fewer arguments than the missing parameters are assigned the value undefined. When we are trying to access an object property that doesn't exist than it will returns undefined.

Q2: What do you mean by REST API?:

REST API is a programming interface, which is used to create web services. It defines the way the web component sends and receives data to the server. The full name of the REST API is Representational State Transfer Application Programming Interface. It is a protocol that makes the journey of data and requests between web servers and clients nice and easy.It is use for standard HTTP conventions for sending and receiving data. It mainly focused to provide a simple and single interface for interacting with data resources, so that data can be created and modified in specific ways.
